# Principle-Learning Rulebook for Battery Safety MCQ Design

---

## 1. Stem Construction Rules

1. Specify the exact abuse or operating condition in the stem (e.g., penetration speed, extrusion force range, ambient pressure level, charging rate, SOC value) before asking about outcomes; never leave the loading scenario implicit.

2. Name the cell format, chemistry, and capacity when they are causally relevant to the answer; omit them only when the question tests a chemistry-independent principle.

3. State the measurement or observable variable being monitored (e.g., surface temperature, voltage trace, internal pressure, gas composition) and explicitly note what is not directly observable (e.g., internal short-circuit location, local temperature at the nail tip).

4. When the question involves a threshold or transition, describe the threshold definition precisely in the stem (e.g., "defined as voltage loss after rest and absence of thermal runaway") so that answer discrimination depends on understanding the phenomenon, not on guessing the definition.

5. For multi-mechanism scenarios, list all active mechanisms in the stem preamble (e.g., mechanical damage, thermal heating, electrochemical side reactions) and ask which combination or ranking is correct, forcing the respondent to weigh competing pathways.

6. When the stem introduces a protective or modified component (PCM, coated separator, additive, surface layer), state its intended function and one confirmed performance attribute before asking about its limitations or secondary effects.

7. For reaction-identification questions, explicitly scope the answer space with a boundary condition stated in the stem (e.g., "exclude gas-phase radical termination reactions," "exclude combustion oxidation steps") to prevent answer ambiguity across reaction classes.

8. For module- or pack-level questions, specify the spatial arrangement, electrical connection topology, and inter-cell spacing so that propagation reasoning is grounded in a concrete geometry.

---

## 2. Evidence Transformation Rules

1. Convert a finding that contains both a local measurement and a system-level conclusion into a question that provides only the local measurement and asks which system-level inference is valid; place the unsupported inference as a distractor.

2. When a source reports a non-monotonic trend (e.g., temperature first rises then falls with increasing pressure), transform it into options that include the monotonic version and the non-monotonic version as competing choices; never reveal the direction of the non-monotonicity in the stem.

3. When a source distinguishes a triggering factor from a dominant heat-generating factor, write separate questions for each role; do not merge them into a single option that conflates the two roles.

4. When a source reports that two variables interact (e.g., SOC and penetration depth both affect outcome but with different sensitivity), construct the question so that single-variable reasoning selects the wrong answer and only joint reasoning selects the correct one.

5. When a source reports a structural or microscopic observation (e.g., delamination sequence, interface adhesion ranking, Li-ion concentration profile), transform it into a question about the observable consequence or the causal ordering rather than simply asking the student to recall the observation.

6. When a source reports a model calibration result, transform it into a question about whether calibration equivalence implies physical correctness, so that the question tests inference validity rather than factual recall.

7. When a source reports a protective strategy (e.g., PCM, fuse, CID), transform its limitation or boundary condition into a question option; confirmed effectiveness and confirmed failure mode should both appear in the option set.

8. When a source reports a gas-generation reaction, categorize reactions by their product type and ask the student to classify a set of candidate reactions; ensure that reactions from adjacent categories (e.g., CO vs. CO₂ producers) appear together as distractors.

---

## 3. Distractor Design Rules

1. Create one distractor by asserting that a surface or external measurement (temperature, voltage) accurately reflects the internal quantity it is commonly assumed to represent; pair it with a correct option that states the measurement underestimates the internal value.

2. Create one distractor by extending a result that is true under one loading regime (e.g., low extrusion force) to all loading regimes; the correct answer restricts the result to its valid domain.

3. Create one distractor by asserting a monotonic relationship where the source reports a non-monotonic one (e.g., "maximum temperature rises continuously with increasing pressure" when it actually peaks then declines).

4. Create one distractor by stating that a protective mechanism (e.g., PCM, fuse, CID) is completely effective under all conditions; the correct answer specifies the condition under which it fails or is insufficient.

5. Create one distractor by attributing the dominant heat source to a secondary reaction (e.g., cathode decomposition when the correct answer is cathode-electrolyte reaction, or ISC heat when it is actually only a trigger).

6. Create one distractor by asserting that model calibration agreement with experimental data implies physical correctness of the model's assumptions.

7. Create one distractor that confuses a transient local event with a sustained system-level state (e.g., "thermal runaway has onset whenever local temperature exceeds the threshold" versus "the system can recover from repeated threshold exceedances").

8. Create one distractor by asserting that a structural reinforcement (e.g., stronger bottom plate) unconditionally reduces risk, when the correct reasoning shows it can redirect loads and increase risk at a different site.

9. For reaction-classification questions, include at least two distractors that are chemically plausible reactions of the same reactant but produce a different product species than the one asked about; these exploit partial chemical knowledge.

10. For sensor and detection questions, create one distractor asserting that conventional signals (temperature, pressure) are equivalent to or earlier than chemical gas signals in warning time.

---

## 4. Reasoning Enforcement Rules

1. Require the respondent to apply at least two interacting physical mechanisms to reach the correct answer (e.g., contact resistance affects current magnitude, which affects local heat generation, which determines whether runaway threshold is crossed); single-mechanism reasoning must select an incorrect option.

2. Include at least one option that is true in isolation but becomes false when a stated boundary condition is applied; correct answering requires recognizing the boundary condition stated in the stem.

3. For multiple-select questions, set the correct answer set so that selecting all options is always wrong; include at least one plausible-sounding but incorrect option to penalize over-inclusion.

4. For questions about protective materials or design modifications, require the respondent to distinguish between the primary intended mechanism and a secondary or unintended consequence; an option describing only the primary mechanism without its limitation should be incorrect.

5. For questions about measurement observables versus internal states, require the respondent to reason about spatial or temporal gaps between the observable and the actual phenomenon (e.g., surface temperature lags internal temperature; voltage cannot detect localized internal events reliably).

6. For questions about model validity or experimental interpretation, require the respondent to distinguish between empirical fit and mechanistic validity; fitting experimental data is not sufficient evidence for physical correctness.

7. For questions involving thresholds or phase transitions, require the respondent to identify whether a described condition exceeds, approaches, or oscillates around the threshold, rather than simply recalling the threshold value.

8. For questions about propagation or cascade behavior, require reasoning about the direction and magnitude of heat or mass transfer between adjacent components, not just the state of the triggering cell.

---

## 5. Difficulty Control Rules

1. Set baseline difficulty by asking about a single observable outcome under a single well-defined condition; increase difficulty by adding a second interacting variable or a counter-intuitive boundary condition without changing the stem format.

2. Control difficulty by varying the number of correct options in multiple-select questions: two correct options from six produce lower difficulty than four correct options from seven, because the larger correct set requires more precise discrimination of each option.

3. Increase difficulty by placing the critical discriminating detail in an option rather than in the stem; easier versions embed the condition in the stem and ask for the consequence.

4. Increase difficulty by requiring the respondent to identify a correct negative result (e.g., a structural reinforcement that increases rather than decreases risk, a protective material that aggravates a secondary hazard) rather than a positive outcome.

5. Control difficulty by varying the specificity of reaction-level knowledge required: identifying a reaction class (e.g., SEI decomposition) is easier than identifying the specific product species (e.g., CO₂ versus CO versus H₂) from a given reactant under a stated condition.

6. Increase difficulty by introducing a confound between correlation and causation: provide two variables that co-vary and ask which is causal; distractors assert the correlated but non-causal variable.

7. Reduce difficulty by providing explicit quantitative thresholds or reference values in the stem; increase difficulty by omitting these values and requiring the respondent to infer the direction of an effect from first principles.

8. For the hardest items, combine a non-monotonic trend, a boundary-condition restriction, and a measurement-versus-internal-state gap in a single question; each factor independently eliminates a subset of distractors, but all three must be applied simultaneously to select the complete correct answer set.
